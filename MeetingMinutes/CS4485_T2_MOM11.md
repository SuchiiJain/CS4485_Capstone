## Discussion Summary

### 1. Review of Previous Action Items

Week 10 action items were largely completed. The new frontend UI was successfully deployed. Auto-fix functionality was merged to both the frontend and backend repositories. Remaining work is focused on bug fixes, polish, and final demo preparation.

### 2. Frontend UI Bug Fixes and Polish

Several bugs identified during Week 10 testing were resolved:
- The Issues page was fixed to display only flags associated with the selected scan, rather than all flags across all scans
- The rot gauge percentage display was centered correctly
- The sidebar user settings/account/sign-out button visibility on the dashboard was fixed
- A popup was added to the "+ Add Repo" button detailing setup steps for users
- Closed issues now correctly improve the rot score
- The ability to reopen closed issues was added
- Redundant UI elements (search bar, extra buttons) were removed from page headers

### 3. Auto-Fix Feature Status

Auto-fix was fully merged to both the frontend and backend repositories. The feature currently supports individual issues on the scan issues page. Extending auto-fix support to AI suggestions remains an upcoming task.

### 4. Local Development Environment Improvements

A change was made to allow the frontend to run locally in preview mode without requiring Firebase `.env` variables to be present. This only applies when running `npm run dev` — production builds via `npm run build` still require valid environment variables. The change does not affect deployment or live functionality.

### 5. Remaining Tasks and Final Demo Prep

- Spec file needs to be updated to its final version for submission
- Further frontend polish and responsiveness fixes ongoing
- PR review process reaffirmed — all changes should go through a PR or team review before merging to main, except minor updates like status reports

### 6. Final Submission Readiness
