## Discussion Summary

**1. Action Item Follow-Ups from MOM #5**  
Most action items from MOM #5 were addressed. The database has been migrated from SQLite to Supabase. The spec file update and hosting/deployment research are in progress. The React dashboard has been made more interactive with drill-down navigation.

**2. Database Migration: SQLite → PostgreSQL (Supabase)**  
Team led the migration of the database layer from SQLite to Supabase-hosted PostgreSQL. The SQLite implementation had caused file-locking issues and could not support production use since it required restarting both the API server and frontend after every scan in order to reflect new data.  
The database was migrated using `psycopg2` and a `DATABASE_URL` environment variable. DNS connection issues were encountered on some networks (notably school Wi-Fi); the team resolved this by using Supabase's pooler connection string format.  
The database write was initially only connected to `run.py` (the test runner) rather than the full production pipeline, so this gap was identified and is being resolved.

**3. Frontend Interactivity and Dashboard Updates**  
Team updated the frontend auth UI with a cleaner login and sign-up design, removed unused placeholder pages from the side navigation, and made dashboard components interactive with drill-down navigation to detail screens.  
The Issues, Projects, and Scan History pages were enhanced with more interactive detail views. Full frontend integration is pending the finalized backend/database version.

**4. Authentication Implementation**  
GitHub OAuth authentication was implemented on a dedicated backend branch. This included:
- `backend/auth.py` with OAuth login and callback routes  
- JWT middleware  
- Protection of the `/api/scan` endpoint  

Frontend updates included:
- `AuthContext.tsx`, `LoginPage.tsx`  
- Updates to `client.ts` and `App.tsx`  

The full OAuth login and logout flow was successfully tested locally. A noted issue was that the GitHub integration account (`5enyl`) lacks write access to both repos, requiring manual commits.

**5. AI Integration Research**  
Team began brainstorming and implementing the AI/LLM integration on a new branch called `AI-integration`. This includes exploring how AI suggestions can be inserted into the existing pipeline after deterministic mismatch detection.

**6. Spec File Update and Hosting/Deployment Research**  
Team is working on updating the spec file to reflect the Supabase migration, updated file structure, and Phase 3 progress.  
Research into hosting and deployment strategy is also in progress, evaluating options including:
- AWS + Docker (original plan)  
- Render (backend) + Vercel (frontend) + Supabase (database)

**7. In-Meeting Discussion**  
The team discussed the current state of the project and progress toward completing the MVP. It was agreed that the backend is largely functional, while the frontend is still under development and requires further work to improve usability and integration.
Supabase has been set up for both database and authentication purposes. The team discussed using GitHub OAuth as the primary login method, potentially integrating it with Supabase authentication. However, additional work is still needed to fully connect the database to the GitHub Actions workflow so that scans and results can be processed automatically.
The TA emphasized the importance of deploying a working version of the system that is accessible to both the professor and TA. While the system can technically be accessed through GitHub, it currently requires manual setup (such as editing YAML files), making it less user-friendly. The team was advised to focus on improving accessibility and simplifying deployment so that non-technical users can interact with the system.
Scalability was also discussed, including handling multiple users, implementing rate limiting, and planning for future system growth. The team agreed that additional documentation should be created outlining future improvements related to scalability and system design.
The TA also asked about the use of the specification document. The team acknowledged that while the spec exists, it has not been consistently referenced during development. The TA recommended using the spec more actively for validation and alignment, especially during testing.
Finally, the team discussed potential AI-related features/tools and how documentation-related features could be incorporated into the frontend in future iterations.
