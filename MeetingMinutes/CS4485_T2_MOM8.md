## Discussion Summary

### 1. Action Item Follow-Ups from MOM #7
Most action items were completed, including integration and deployment progress. Remaining work is focused on UI improvements, bug fixes, and improving feature completeness.

---

### 2. Frontend Bug Fixes and UI Updates
Several frontend issues were addressed:
- Fixed UI inconsistencies and navigation layout
- Updated sidebar and user settings display (including profile avatar)
- Improved scan history panel and issue display formatting
- Corrected success rate and rot score calculations
- Added a resolve/reopen button in the issue table as part of frontend preparation for issue resolution functionality

Remaining work includes final UI polish and ensuring consistent behavior across all pages.

### 3. Scan Baseline Issue
A bug was identified where new scans were incorrectly creating a new baseline instead of comparing against the previous one.
- Fix implemented to ensure scans now compare against the latest stored fingerprint
- Manual reset of baseline in Firestore was required during testing
- System now correctly tracks incremental changes across scans

---

### 4. Repository Filtering Issue
A frontend bug caused all repositories to display instead of only user-specific repositories.
- Issue identified during testing
- Partial fixes applied, but inconsistencies remain after deployment
- Further debugging required to ensure correct filtering

### 5. Backend–Frontend Data Linking

- Scan results and issues now display correctly in frontend

- Some inconsistencies remain between local and deployed versions
