## Discussion Summary

### 1. Action Item Follow-Ups from MOM #7
Most action items were completed, including major integration work and deployment progress. The system is now closer to a fully functional demo, with remaining work focused on bug fixes, UI polish, and deployment consistency.

---

### 2. Frontend Bug Fixes and UI Updates
Several frontend issues were identified and addressed:
- Fixed UI inconsistencies and navigation layout
- Updated sidebar and user settings display (including profile avatar)
- Improved issue display and scan history panel
- Fixed success rate and rot score calculations

Remaining UI work includes final polish and ensuring consistent behavior across pages.

---

### 3. Scan Baseline Issue
A major bug was identified where new scans were incorrectly creating a new baseline instead of comparing against the previous one.
- Fix was implemented to ensure scans now compare against the most recent stored fingerprint
- Manual reset of baseline in Firestore was required during testing to clear stale data
- System now correctly tracks incremental changes across scans

---

### 4. Repository Filtering Issue
Frontend bug caused all repositories to display instead of only user-specific repositories.
- Issue identified during testing
- Partial fixes attempted, but inconsistencies still observed after deployment
- Requires further debugging to ensure correct filtering based on authenticated user

---

### 5. Backend–Frontend Data Linking
- Fixed Firestore structure mismatch (issues vs flags collections)
- Scan results and issues now correctly display in frontend
- AI suggestions are now being stored in Firestore and partially integrated into frontend
- Some deployment inconsistencies remain between local and hosted versions

---

### 6. Demo Readiness and Testing
- Core pipeline is functional end-to-end
